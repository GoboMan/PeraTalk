# LLM-API 方針（英会話・将来の多言語拡張）

[← README に戻る](../../README.md)

> 本ドキュメントは **LLM のプロバイダ・既定モデル・呼び出し経路の単一ソース**。他ドキュメント（README／会話／インフラなど）は**ここを参照**するに留める。

本アプリでは推論（会話相手・文法・言い換え・セッション後のフィードバック／ボキャブラリ候補の生成など）を **Google** の **Gemini** モデルで行う。**初版**は会話・フィードバックの**主言語を英語**にそろえ、プロンプトと評価の負荷を抑える。**将来的に**他言語の学習ペアを足す場合も、同じ API 枠・テンプレ構造を流用する想定。

**呼び出し経路**：開発・試行・キー発行は **Google AI Studio**（**Gemini API**）を前提とする。**本番**では、請求・データ所在地・運用ポリシーに応じて **Vertex AI**（**Google Cloud** 上の Gemini）へ寄せる選択肢がある（同一 Gemini ファミリーの利用が一般形）。

端末にウェイトを同梱・追加ダウンロードしないため、**バッテリー消費・ストレージ・初回データ量**をローカル LLM ほど圧迫しない（[インフラ-Supabase](インフラ-Supabase.md)）。**コスト**と**構造化出力**を重視した方針を以下にまとめる。

---

## 0. 製品方針（既定・拡張）

| 項目 | 方針 |
|------|------|
| **プロバイダ** | **Gemini**（**Google**）。利用開始・試行は **Google AI Studio**（**Gemini API**）。本番は **Vertex AI**（**Google Cloud**）で同系モデルを呼ぶ構成も含めて検討する（**API キーはモバイルに埋め込まず**バックエンド／BFF に限定）。 |
| **既定モデル（検討中）** | **Gemini 3.1 Flash Lite**。API 上のモデルコードは現時点で **`gemini-3.1-flash-lite-preview`**（プレビュー。GA や名称変更時は [モデル一覧](https://ai.google.dev/gemini-api/docs/models/gemini-3.1-flash-lite-preview) を優先して本書を更新する）。 |
| **クライアントからの直叩き** | **API キーはモバイルアプリに埋め込まない。** バックエンド（または BFF）が代理で呼び出し、認証・レート制限・利用量の集約を行う想定。 |
| **構造化出力** | セッション終了後の **総括フィードバック・ボキャブラリ候補** などは、Gemini API の **Structured outputs**（`response_json_schema` 等）でスキーマに沿わせ、パースと検証コストを抑える（失敗時のリトライや短いフォールバックは実装で定義）。 |
| **ユーザーからのモデル指定** | **不可**。クライアント（設定画面を含む）から **Gemini のモデル ID を選ばせない**。実際に呼び出すモデル（**Flash Lite** のまま／**Flash** や **Pro** へ変更／プレビューから安定版への移行など）は**バックエンド・運用設定およびデプロイのみ**で決める。会話の文脈・終了後処理の流れは [会話](../機能/会話.md) を参照。 |

## 1. 想定している負荷（プロダクト由来）

| 用途 | 内容 | API 利用上のポイント |
|------|------|----------------------|
| **インタラクティブ** | **初版は英語**での相手役の発話、短い文脈の維持 | **往復ごとのリクエスト数**・レイテンシ。ストリーミングは UX 次第で検討。 |
| **教育系フィードバック** | 誤り指摘、言い換え、場面に合う表現 | 指示テンプレの一貫性。**英語（会話）**と**説明・補助言語**（例：日本語）を明示。 |
| **セッション後処理** | **総括**の講評・**ボキャブラリ候補**など、**構造化されやすい長めの出力** | **出力トークン量**が課金に効く。スキーマで無駄な前置きを減らす。 |
| **将来：蓄積に基づく追加フィードバック** | 蓄積された会話・総括・ログ等を入力に、ユーザー依頼で**追加の講評や傾向**を返す | **入力コンテキストが伸びやすい**。ウィンドウ・要約・キャッシュ（利用可能なら）と課金試算が必要。詳細は [会話](../機能/会話.md) の将来検討を参照。 |
| **将来：多言語拡張** | 学習言語を英語以外にも広げる場合 | プロンプトで言語を固定し、**回帰用の短い評価セット**を言語別に持つ（初版は英語セットから開始）。 |

---

## 2. 選定の軸（優先順位の目安）

1. **コスト**：**Flash Lite** 帯を既定にし、プロンプト長・会話履歴の **ウィンドウ管理**（要約・トリム）と **コンテキストキャッシュ**（利用可能）でトークンを抑える。
2. **構造化出力**：フィードバック系は **スキーマ付き応答** を優先する（[モデル仕様上 Structured outputs 対応](https://ai.google.dev/gemini-api/docs/models/gemini-3.1-flash-lite-preview)）。
3. **プライバシー・運用**：送信データの範囲（保持する会話ログの粒度）、**データ保持ポリシー**・利用地域は **Google AI Studio（Gemini API）** または **Vertex AI** のいずれを採用するかに応じた利用条件・プロダクトのプライバシーポリシーと揃えて文書化する（詳細は実装フェーズで確定）。**サーバー DB に載せる会話由来テキストは記憶用要約（サニタイズ済み）のみ**とし、ターン原文は端末に留める境界は [会話](../機能/会話.md) §5 が正。

---

## 3. プロンプトと実装のプラクティス

- **テンプレ**：**初版は「英語＋説明・補助言語」＋難易度**を**毎回明示**する。将来、学習言語が増えても**同じ枠**に載せる。モデル ID を変えてもテンプレ構造は揃える。
- **後処理**：候補の **Kind** はスキーマに近い形で返す。**パース失敗**時はリトライや短いフォールバックを設計する。
- **Thinking（該当モデル時）**：AI Studio／API で思考量を調整できる場合は、**高頻度チャットは低め**、**終了後のまとめ生成は必要に応じて上げる**など、コストと品質のバランスを実測で決める。
- **監査**：開発時はログに **リクエスト／トレース ID** を残し、不具合調査とコスト突合しやすくする。
