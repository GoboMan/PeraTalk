# 音声入力フェーズ（A／B／C）

[← README に戻る](../../README.md)

会話機能の**入力／音声まわり**を、実装の重さ別に **A／B／C** の 3 段階に切り分ける。本ドキュメントを**単一ソース**とし、他ドキュメント（[会話](../機能/会話.md)、[会話-ペルソナとTTS](../機能/会話-ペルソナとTTS.md)、[将来-リアルタイム音声](../ロードマップ/将来-リアルタイム音声.md) ほか）からはここへリンクする。

---

## 1. フェーズ定義と今フェーズのスコープ

**今フェーズ（初期リリース）に含めるのは A と B**。**C は後フェーズ**で検討する。

| レベル | 内容 | 今フェーズのスコープ |
|--------|------|----------------------|
| **A. テキスト** | タイピング入力。LLM 往復・読み上げは既存フローのとおり。 | **含める** |
| **B. ターン制の音声** | ユーザーが**一区切り話す／録音を確定** → **音声認識（ASR）** でテキスト化 → LLM → **表示＋読み上げ（TTS）**。往復ごとにターンが切り替わる。割り込みや常時ストリームは必須としない。 | **含める** |
| **C. リアルタイム音声会話** | **常時マイク**に近い**低遅延の双方向ストリーム**、**話し始めでの割り込み（バージイン）** など。**ChatGPT の音声会話モード**や [**Gemini Live API**](https://cloud.google.com/vertex-ai/generative-ai/docs/live-api)（Vertex AI のセッション型・WebSocket／ネイティブ音声）級の体験。仕組み・料金の整理は [将来-リアルタイム音声](../ロードマップ/将来-リアルタイム音声.md) を参照。 | **今は含めない**（別フェーズで設計・実装） |

---

## 2. B で目指す体験（補足）

- 「日本語で聞いたら日本語で答え、英語で話したら英語で返す」などの**言語の切り替え**は、**B の範囲で**システム指示および（任意で）発話単位の言語判定で実現する想定。**C と切り離して**載せられる。
- B ではオーディオの**状態管理・認識品質・端末／Bluetooth**などの検証が主な増分。**C が必要となる**長時間 WebSocket／プロキシ基盤／ストリーム同期は、初期リリースのスコープ外とする。
