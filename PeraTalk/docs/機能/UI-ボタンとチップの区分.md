# UI — ボタンとチップの区分

[← 機能一覧に戻る](機能一覧.md) ／ [← README に戻る](../../README.md)

画面上で **データの変更（CRUD）** と **表示・分類・絞り込み** を、**形**と**塗り**で一目で区別できるようにするための方針。

---

## 意図

| ユーザーの理解しやすさ | デザイン上の対応 |
|------------------------|------------------|
| **この操作で DB（サーバー）へ書き込み／削除が発生する** | **角丸四角** かつ **背景をベタ塗り**（ソリッドフィル） |
| **画面ローカルだけで状態が変わり、確定保存まで DB に書かない** | **角丸四角** だが **ベタ塗りにしない**（線枠・ゴースト・透明＋枠など） |
| **種類・タグ・状態の表示やフィルタに過ぎない** | **左右が丸いピル型チップ**（現行タグと同一ファミリー） |

---

## A. CRUD・確定・送信系 — 「角丸四角ボタン」

- **形状**：十分な角丸の **四角形ボタン**（完全ピルにしない）。角丸半径はデザイントークンで **タグピルより角が弱く**見えるようにし、二者が混ざらないようにする。
- **塗りと永続化の対応（重要）**
  - **背景ベタ塗り（ソリッド）**：その操作のタイミングで **DB（同期先）への書き込みまたは削除が発生する**ものに使う。例：**保存**、**ログアウト**、一覧での **即時削除**、モーダルの **確定してサーバーへ反映する OK** など。
  - **ベタ塗りにしない（アウトライン／セカンダリ／ゴースト）**：**画面上の編集中モデルだけ**が変わり、**このボタンだけでは DB にまだ書かない**ものに使う。例：**例文を追加**・**例文ブロックの削除**（**保存**するまで永続化しない）、下書きの **品詞ブロック追加**、単語追加／編集フォームの **AI生成**（見出し語入力後に品詞・意味・例文を一括でドラフトへ流し込む）、フォーム内の **キャンセル可能な追加操作** など。
- **バリアント**：Primary（ベタ塗り＝永続反映）／Outline・Ghost（非ベタ塗り＝ローカル編集）／Destructive／Disabled はトークンで分ける。**形状クラス（角丸四角）は共通**とする。

---

## B. 種類・タグ・フィルタ — 「フルピル（タグ型チップ）」

- **形状**：**左右が完全に丸いチップ**（一覧フィルタ・タグ付けで既に使っているピルと同系）。
- **用途の例**：エントリに付いた **タグ**、一覧の **Kind／フォルダ／状態のフィルタ**、**プリセットラベルの選択表示**など、「**ラベル／カテゴリ**」を表すもの。
- **データとの関係**：フィルタは **表示の切り替え** が主目的。**タグの付与・解除**もピルでよいが、**一覧や詳細の確定保存**が別にある場合は、最終的な **書き込みは A の保存ボタン** に寄せるとルールがぶれにくい。

---

## C. 編集スコープの切り替え — タブ／セグメント（A・B とは別コンポーネント）

- **用途**：例：**用法（品詞）タブ**のように、「どのブロックを編集しているか」を切り替える。
- **形状**：**A の角丸四角にも B のピルにもしない**。下線タブ・セグメント等の **専用パターン** を使い、**保存ボタン（A）と誤認**しないようにする。

---

## 対応表（ざっくり）

| 性質 | 形状 | 塗り | 例 |
|------|------|------|-----|
| 確定して DB へ反映 | 角丸四角 | **ベタ塗り** | **保存**、即時削除、送信 |
| 編集中モデルのみ変更（未保存） | 角丸四角 | **非ベタ塗り** | **例文を追加**、例文ブロック削除、（保存前の）品詞ブロック追加、**AI生成**（単語追加・編集） |
| 分類・フィルタ・タグ表示 | フルピルチップ | （チップ規則に従う） | タグ、Kind フィルタ |
| フォーム内のスコープ切替 | タブ／セグメント | （専用） | 用法タブ |

---

## Figma・実装メモ

- コンポーネント命名の例：**`Button / Action / Persist`（ベタ塗り・DB 反映）** と **`Button / Action / Local`（非ベタ塗り・編集中のみ）** と **`Chip / Tag or Filter`（ピル系）** を分離する。
- 角丸半径・高さ・パディングは **変数化**し、A と B で値が入れ替わらないようにする。
- **同じラベルでも意味が違う場合**は塗りで分ける（例：「削除」が **DB から即消す**ならベタ塗り／Destructive、**フォーム上の行だけ消す**なら非ベタ塗り）。
