# 初版スコープ（Phase 1）

[← README に戻る](../../README.md)

初回リリース（Phase 1）で**含むもの／含まないもの**を [機能一覧](../機能/機能一覧.md) へのリンク集として整理する。**機能の詳細仕様はここには持たない**（二重記述を避けるため、各機能ファイルを正とする）。

**想定ユーザー**：**英語の中上級者**を主ターゲットとし、会話での**アウトプット中心**の学習にフォーカスする（動機・プラトーと想起の整理は [ビジョンと目的](../概要/ビジョンと目的.md) の「ターゲットユーザー」「中上級者の課題と、本アプリが『刺さる』理由」、[学習サイクル](../概要/学習サイクル.md) を参照）。

## 差別化アイデア（Phase 1 外・候補）

以下は**初版では実装しない**が、中上級者向けの差別化としてメモする。仕様化するときは該当機能ドキュメントを新設または更新する。背景の整理は [ビジョンと目的](../概要/ビジョンと目的.md) の「中上級者の課題と、本アプリが『刺さる』理由」。

| アイデア | 概要 |
|----------|------|
| **言い換えチャレンジ** | 会話中に *very* / *good* 等の「使わない語」を指定し、**Apple Intelligence（オンデバイス）**が提示する**類義・より洗練した語**を使うことを促すモード等。 |
| **文脈タグ（記憶のフック）** | 記録時に「カフェの話題で出た」等の**場面タグ**をオンデバイスで付与し、後から思い出しやすくする。 |
| **自動テスト（オンデバイス生成）** | 記録語から穴埋め・短い英作文などを **Apple Intelligence 等のオンデバイス経路で生成**。クラウド LLM を消費しないため、**回数を寛く設計しやすい**（既存 `QUIZ` 行との関係は後フェーズで整理）。 |

## リリース基準（暫定）

- Phase 1 のすべての機能で**受け入れ条件**（各機能ファイルの該当節）を満たすこと。
- [収益とサブスク](../運営/収益とサブスク.md) の **プラン別機能一覧**（無料／プラス／プロ。店頭価格は **プラス・プロ**列見出しセル内の月額・年額）どおりに StoreKit 上で課金が動作すること。
- 認証経路（[設定とアカウント](../機能/設定とアカウント.md)）が一通り疎通していること。

> 受け入れ条件の詳細・テスト方針は実装フェーズで各機能ファイルに追記する。
