# 将来-キャラ音声（アニメ風ボイス・ローカル音声合成）

[← README に戻る](../../README.md)

---

**ステータス**

- **初回リリースでは実装しない。** 収益・体験の伸びしろとして、調査結果を記録のみしておく。

**技術的な方向性（概要）**

- **パイプライン例:** LLM がテキスト応答を生成する → **標準 TTS** で一度読み上げる → **RVC（Retrieval-based Voice Conversion）系**などで声質変換して出力する、という構成が検討候補。
- **オンデバイス:** チップ世代・モデル選定次第だが、`Pocket TTS` や `Kokoro-82M` など**軽量 TTS／推論スタックをローカルに載せる**方向は、遅延と通信なしの両面で現実的になりつつある、という整理（名称・推奨構成は時点で更新する）。

**権利・審査（最大の制約）**

- **第三者の特定アニメ・声優さんの声を無許可で学習／配布し商業利用する**構成は、**パブリシティ権・著作権・キャラ権利・プラットフォーム規約**のどれでも踏みやすく、App Store 審査とも相性が悪い。**「できる」≠「製品として安全」**。

**現実寄りの打ち手（概要）**

- **案A・オリジナル：** キャラごと実在作品に寄せすぎず、許諾済みの**オリジナル収録またはライセンス付き音声**で「アニメ調」「ツンデレ調」などを表現する。
- **案B・ユーザー責任モデル：** アプリはプレイヤー／パイプラインのみ提供し、**ユーザーが自分で用意したモデルを読み込む**。開発者側リスクは下がっても審査・説明責務は残る前提。
- **案C・公式コラボ：** 規模が出てから IP 側や VTuber と**追加ボイスパック販売**などを契約ベースで行う。

**まとめ（このメモの位置づけ）**

> 好みの「アニメ声」での会話練習は、将来プロダクト価値になり得るが、**初版は標準音声＋プロンプト上のキャラ設定**など（ペルソナ・TTS の具体は [会話-ペルソナとTTS](../機能/会話-ペルソナとTTS.md)）、権利的に筋の通った形から進め、音声クローン級の体験は上記のような整理のうえで後追い検討とする。
