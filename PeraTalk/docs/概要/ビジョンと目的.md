# ビジョンと目的

[← README に戻る](../../README.md)

PeraTalk が**何のためのアプリで、誰の何を解こうとしているか**を、著者の体験を起点に整理する。製品コンセプトの差別化（人間らしい記憶・パートナー分離）の詳細は [コンセプト-人間らしい記憶](コンセプト-人間らしい記憶.md) を参照。

---

## プロダクトの一行まとめ

- **iOS** 向けの**英会話**（英語の学習・練習）アプリ。
- **初版**は会話・単語帳・LLM 向け指示の**主言語を英語**にそろえる。**将来的には**他言語ペアへの拡張（学習言語の選択）も視野に入れ、データモデル・プロンプト枠は**伸ばしやすい形**で置く。
- **ボキャブラリーブック**（[単語帳](../機能/単語帳.md)）と**会話**（[会話](../機能/会話.md)：Self／AI）をプロダクトの中心とする。
- 自分の語彙を**文として使える力**と、**場面に応じた表現の引き出し**を伸ばすことをねらう。
- [学習サイクル](学習サイクル.md)（アウトプット → フィードバック → インプット）を、会話機能で回す前提。

---

## 開発の動機（著者の体験・英語の例）

初版は**英会話**にフォーカスするが、設計の動機としての**著者の言語経験**を以下に残す（リスニングや語彙の例はいずれも**英語**の話）。

**背景・現状**

- 外国語学部・英米語学科。4 年間まるまる英語で学んだ。
- 外国人とのコミュニケーションは一通り取れるが、ネイティブ並みとは言い難い。

**理解（リスニング・読解）と産出**

- リスニングはディズニー映画級の英語で、内容の **70〜80%** 程度は聞き取れるイメージ。
- 聞き取れた内容を自分の文として組み立てる力は弱く、**話す・書くは理解より下**に感じる。
- **語彙**はディズニー級と同程度かそれ以上もあり、他スキルより相対的に強い。
- 一方、**文に組み立てる力**とのギャップが大きい。

**つまずき**

- 単語の意味・ざっくりした用法はわかるが（例：*miserable*）、**その語を使った文**がすぐ出ない。
- **フレーズの引き出し**が少なく、場面に合う言い返し・言い回しをすぐ選べない。

**うまくいっている点**

- 単語の寄せ集めではなく、**文の構造（SVC / SVO / SVOO など）**を踏まえてリスニングで追えるようになった。

**優先したいこと**

- **表現力**の拡大。
- **持っている語彙を文にする力**の拡大。
- **場面に応じたフレーズの引き出し**の拡大。

## 目的（このアプリで実現したいこと）

- **理解と産出のギャップ**（前節は英語での体験例）を、**英語のアウトプット**と同じ仕組みで補う。  
  **アウトプット（話す／書く）→ フィードバック → インプット（見返し・単語帳）** のループで回す（抽象モデルは [学習サイクル](学習サイクル.md)）。将来、別の学習言語を追加するときも同じループの使い回しを想定する。
- **単語帳**でインプットを蓄え、**会話**でアウトプットと AI からの修正案を受け取り、セッション後に**ボキャブラリ候補**として単語帳へ載せる一歩手前までまとめる（詳細は [機能一覧](../機能/機能一覧.md)）。
- バックエンドは **Supabase** で **単語帳**および **記憶用セッション要約（サニタイズ済み）** などを同期する（**会話ターン本文はサーバーに載せない**。[インフラ-Supabase](../アーキテクチャ/インフラ-Supabase.md)、[会話](../機能/会話.md) §5）。
- **推論**および**既定モデルの選定**は [LLM-API方針](../アーキテクチャ/LLM-API方針.md) を**単一の正**とする。**音声**は端末側 TTS 中心（[会話-ペルソナとTTS](../機能/会話-ペルソナとTTS.md)）。
- **会話入力**のスコープ（A／B／C）と今フェーズで含める範囲は [音声入力フェーズ](../アーキテクチャ/音声入力フェーズ.md) を参照。
- **会話相手（Persona）6 パターンと TTS の方針**は [会話-ペルソナとTTS](../機能/会話-ペルソナとTTS.md) を参照。

## コンセプト・差別化（要約）

他の英会話アプリとの違いとして、**できるだけ AI っぽさを減らす**ことを前面に出したい。**連続した対話をしている感じ**（相手が話を覚えているように見えること）と、**現実の人間関係に近い「知らない相手には説明が必要」さ**の両方を、両立させるのが柱になる。

- 表面的には「**記憶している**」ように見せる：同じパートナーとの会話では、毎回ゼロから同じ説明を繰り返さなくてよい体験に寄せる。
- 裏側は **RAG**：**記憶用要約**をベクトル化して保存し、応答時に必要な情報だけ retrieve してプロンプトへ合成する。
- AI パートナーを**人単位で分け、記憶も分ける**：パートナー A にだけ話した内容は、B には知られていないように見せる。
- **モチベーション**：自分の話を相手が覚えているように感じられると、関係性への好意・話したい気持ちが強まりやすく、英語のアウトプット量・継続にも結びつく、という実装理由がある。

詳細（ねらい・トレードオフ・実装上の含意）は [コンセプト-人間らしい記憶](コンセプト-人間らしい記憶.md)。
