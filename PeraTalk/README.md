# PeraTalk

- **iOS** で開発している**外国語学習**アプリ。ユーザーが選ぶ**学習言語**（英語・韓国語・スペイン語など）と**母語（または説明用の言語）**の組み合わせで使える前提で設計する。
- **ボキャブラリーブック**と**会話（Self／AI）**をプロダクトの中心とする。
- 自分の語彙を**文として使える力**と、**場面に応じた表現の引き出し**を伸ばすことをねらう（どの学習言語でも同じ学習サイクルを回せるイメージ）。
- [実現したい学習サイクル](docs/learning-cycle.md)（アウトプット → フィードバック → インプット）を、会話機能で回す前提。

---

## 開発の動機（著者の体験・英語の例）

プロダクトは**英語専用ではない**が、設計の動機としての**著者の言語経験**を以下に残す（リスニングや語彙の例はいずれも**英語**の話）。

**背景・現状**

- 外国語学部・英米語学科。4年間まるまる英語で学んだ。
- 外国人とのコミュニケーションは一通り取れるが、ネイティブ並みとは言い難い。

**理解（リスニング・読解）と産出**

- リスニングはディズニー映画級の英語で、内容の **70〜80%** 程度は聞き取れるイメージ。
- 聞き取れた内容を自分の文として組み立てる力は弱く、**話す・書くは理解より下**に感じる。
- **語彙**はディズニー級と同程度かそれ以上もあり、他スキルより相対的に強い。
- 一方、**文に組み立てる力**とのギャップが大きい。

**つまずき**

- 単語の意味・ざっくりした用法はわかるが（例：*miserable*）、**その語を使った文**がすぐ出ない。
- **フレーズの引き出し**が少なく、場面に合う言い返し・言い回しをすぐ選べない。

**うまくいっている点**

- 単語の寄せ集めではなく、**文の構造（SVC / SVO / SVOO など）**を踏まえてリスニングで追えるようになった。

**優先したいこと**

- **表現力**の拡大。
- **持っている語彙を文にする力**の拡大。
- **場面に応じたフレーズの引き出し**の拡大。

## 目的（このアプリで実現したいこと）

- **理解と産出のギャップ**（前節は英語での体験例）を、ユーザーが選んだ**学習言語**でも同じ仕組みで補う。**アウトプット（話す／書く）→ フィードバック → インプット（見返し・単語帳）** のループで回す（抽象モデルは [docs/learning-cycle.md](docs/learning-cycle.md)）。
- **単語帳**でインプットを蓄え、**会話**でアウトプットと AI からの修正案を受け取り、セッション後に**ボキャブラリ候補**として単語帳へ載せる一歩手前までまとめる（詳細は [docs/features.md](docs/features.md)）。
- バックエンドは **Supabase** でテキスト同期、推論や音声読み上げは**ローカル寄り**で設計する（ [docs/infrastructure-supabase.md](docs/infrastructure-supabase.md) ）。**既定は軽量の Gemma 3**、必要に応じ**追加モデル（例：Qwen）をダウンロード**し、**会話途中も含めモデルを切替**可能とする。詳細は [docs/local-llm.md](docs/local-llm.md) と [docs/flow-conversation.md](docs/flow-conversation.md) を参照。
